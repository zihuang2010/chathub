设计要点

  • 行本体零装饰：置顶行不画 pin 角标，不变色，不加 border。视觉负担一次性收在区段头里。
  • 区段头规格：11px / muted 色 / weight 500 / letter-spacing 0.04em / padding 12-14px 顶部、4-6px 底部、16px 左右。
  • 区段间留白：6-8px 的空间隔代替分隔线。普通会话区不需要 header（"非置顶"是默认态，无需命名）。
  • 排序稳定性：置顶区段内部按 updatedAt 倒序；普通区段同样按 updatedAt 倒序。已经在 data.ts 里做了的话，渲染层只需要 partition() 一次。
  • 空状态处理：置顶数为 0 时整个区段头隐藏，列表退化为纯线性。
  • 头像角标卸载：ConversationAvatar 的 pinned prop 仍保留（搜索结果命中置顶项时还有用），但默认列表里不传。

已知改动量

  • ConversationList.tsx：把 conversations.map(...) 改成 partition→渲染 2 个区段；区段头是一个新组件。
  • 虚拟化兼容：Wave 6 用的虚拟化方案需要支持"非等高 item"——区段头作为一个 30px 左右的特殊 item 插进 itemSize 函数即可。如果用的是 @tanstack/react-virtual，estimateSize 直接按 index 分流。
  • 无样式新增：区段头颜色 / 字号 / spacing 都复用现成 token，不引入新 CSS 变量。
