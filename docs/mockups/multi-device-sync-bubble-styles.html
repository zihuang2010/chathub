多端同步消息（messageDirection=3）差异化样式 · 方案对比

场景：客服在手机企微/微信端直接回复了客户，这条消息经多端同步进入本工作台。 它仍属「我方/出站」（右对齐），但来源不是本工作台 —— 质检/管理需要一眼可追溯。

前提（技术）：当前前端模型只有 direction: "in" | "out"，源方向 3 在后端 to_local_direction(3)→2 被抹平两次，前端目前收不到「3」。原始信号完整保留在 sort_key 第 2 段，落地方式 = 后端派生 1 个布尔字段 syncedFromOtherDevice 透传 → 前端 Message 加可选字段 → MessageBubble 加渲染分支。不改 in/out 语义，纯加法、低风险。 下面每张卡都用你们真实的气泡 token 渲染。
